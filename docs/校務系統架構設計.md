# QuEDU 校務系統架構設計

> 設計確認日期：2026-05-06

## 核心概念：Main / Branch 架構

類比 git：每所學校有一個 **main**（全校學生池），班級是從 main 建立的 **branch**（指標，非副本）。

```
super-admin          → 系統營運（學校管理、帳號管理、系統設定）
school-admin         → 校內事務（main 管理、升級工具、班級生成）
teacher（導師/科任）  → 建立/認領班級，從 main 拉學生
student              → PIN 登入
```

---

## Main（全校學生池）

- 範圍：**每所學校一個 main**（per-school tenant，以 schoolId 隔離）
- 資料來源：Firestore `students` collection，schoolId 即為邊界
- 管理者：school-admin（校內全權）、super-admin（跨校系統層）
- 學生欄位：`{ uid, name, pin, schoolId, schoolName, grade, classNumber, seatNumber, classIds[], createdAt, lastSeen, status }`
- `status`：`active`（在學）、`archived`（畢業/轉出）

---

## Branch（班級）

- 班級只存關係，不複製學生資料
- 學生的 `classIds[]` 陣列存放所屬班級 ID
- 寫入方向：老師在班級修改學生資料 → **覆寫 main**
- 老師可修改欄位：**僅限 `name` 和 `pin`**（學籍資料由 school-admin 管）
- 衝突處理：last-write-wins，不鎖定

---

## 班級類型

| 類型 | 建立者 | 說明 |
|------|--------|------|
| homeroom（原班） | school-admin（一鍵生成）或導師 | 依 grade + classNumber 分組 |
| subject（科任班） | 科任老師 | 從全校 main 篩選後拉入 |

---

## 學生加入班級的路徑

1. **school-admin 一鍵生成原班**：依 main 的 grade + classNumber 自動建立班級並寫入 classIds
2. **老師主動拉入**：科任老師從全校列表（可依年級/班級篩選）勾選學生加入科任班
3. **邀請碼**：保留，用途限縮為**跨校試用**場景

---

## 學年升級工具（由 school-admin 操作）

| 步驟 | 操作 | 方式 |
|------|------|------|
| 1 | 全校 grade +1 | 系統批次 |
| 2 | 六年級學生封存 | 系統批次，status → archived |
| 3 | 舊班級封存 | 系統批次，active → false |
| 4 | 新班級重新分配 | 工具輔助：選學生 → 選目標班級 → 確認 |

---

## Cold Start 情境處理

### 情境 1：main 有名單，但導師沒建原班
school-admin 點「從 main 生成原班」→ 系統依 grade + classNumber 分組建立 class 文件 → 導師登入後認領班級。

### 情境 2：外校科任試用，無 main 無原班
科任老師建科任班 → 學生用邀請碼加入 → 學生帳號標記 `trial: true`，schoolId 為空或 trial。
學校正式加入後，school-admin 匯入名單，系統提示合併 trial 學生到 main。

---

## 與現行系統差異

| 面向 | 現行系統 | 新設計 |
|------|---------|--------|
| 角色 | super-admin / teacher | super-admin / school-admin / teacher |
| 學生建立 | 老師批次產生 或 super-admin 匯入 | school-admin 管理 main |
| 學生加班級 | 邀請碼（學生主動） | 老師主動拉入（邀請碼保留給試用） |
| 原班建立 | 老師自己建 | school-admin 一鍵從 main 生成 |
| 學年升級 | 無工具 | 批次工具（grade+1、封存、重新分班） |
| 畢業/轉出 | 無封存 | status: archived |
| 科任選學生 | 從原班名單拉 | 從全校 main 篩選（年級/班級 filter） |
| 試用路徑 | 邀請碼 | 邀請碼 + trial 標記，可後續合併 |

---

## 實作優先順序

1. **school-admin 角色與 main 管理介面**（解除 super-admin 瓶頸）
2. **「從 main 生成原班」工具**（cold start 解法）
3. **老師主動拉學生介面**（科任班建立流程）
4. **學年升級工具**（grade+1、封存、重新分班）
5. **trial 學生合併流程**（低優先）
